A **block** is a working composition of multiple primitives, shipped
with wiring, variations, and an install command. Where a primitive is a
button, a block is the full sign-in card, or the chat surface, or the
KPI grid.

Blocks consume primitives -- they never reinvent them. Adding a block
to your project installs every required primitive at the right
version through `registryDependencies`.

## Blocks vs Components vs Artifacts

| | Components | Blocks | Artifacts |
|---|---|---|---|
| **Scope** | One primitive | N primitives composed | One primitive that consumes an LLM schema |
| **Example** | `Button` | `sign-in` card | `chart` artifact rendering Recharts from JSON |
| **Wiring** | None | Layout + state + handlers | Schema parser + streaming renderer |
| **Install** | `gremorie add rx-button` | `gremorie add sign-in` | `gremorie add rx-chart-artifact` |
| **Status** | Shipping (100 primitives) | Planned | Planned |

## Featured blocks

The first wave of blocks lands in Phase 5. Each carries an anchor below
so the [CLI](/platform/cli) and MCP server can deep-link to it.

<div id="sign-in" />

### Sign-in

Email and password card with an OAuth slot for social providers.
Includes inline validation, error states, and a "remember me" toggle.

<Callout type="info">Status: planned.</Callout>

<div id="dashboard" />

### Dashboard

Sidebar plus KPI grid plus chart strip plus recent-activity table.
Wired to TanStack Query for data; works in App Router and Pages
Router.

<Callout type="info">Status: planned.</Callout>

<div id="chat-surface" />

### Chat surface

`Conversation` + `Message` + `PromptInput`, fully wired to the AI SDK.
Streaming responses, tool-call rendering, attachments, model picker.
This is the flagship Gremorie block.

<Callout type="info">Status: planned.</Callout>

<div id="settings" />

### Settings form

Multi-section settings form with a sticky save bar, validation, and
optimistic updates. Sections are slot-based so consumers add their own
fields.

<Callout type="info">Status: planned.</Callout>

<div id="marketing" />

### Marketing hero

Hero with CTA, feature grid, social proof. Pure layout block; consumes
only `Button`, `Card`, `Badge`, and tokens.

<Callout type="info">Status: planned.</Callout>

<div id="empty-state" />

### Empty state

No data, no results, error, permission denied. One block, four
variants. Each ships with a sensible default icon and an action slot.

<Callout type="info">Status: planned.</Callout>
