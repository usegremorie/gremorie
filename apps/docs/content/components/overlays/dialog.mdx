`Dialog` is the right primitive for **focused decisions or short
flows** that need to interrupt the user's context - confirmations,
single-step forms, detail cards. For longer flows that don't need
full focus, use `Sheet`. For inline contextual content, use
`Popover`.

The dialog ships with overlay, content, header, footer, title, and
description sub-components. The close button is on by default; set
`showCloseButton={false}` when the dialog renders its own dismiss
affordance.

```tsx
import { Button } from "@gremorie/rx-forms";
import {
  Dialog,
  DialogClose,
  DialogContent,
  DialogDescription,
  DialogFooter,
  DialogHeader,
  DialogTitle,
  DialogTrigger,
} from "@gremorie/rx-overlays";

export function DeleteConfirm() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button variant="destructive">Delete project</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Delete project?</DialogTitle>
          <DialogDescription>
            This cannot be undone. All tasks, comments, and attachments will
            be removed.
          </DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <DialogClose asChild>
            <Button variant="outline">Cancel</Button>
          </DialogClose>
          <Button variant="destructive">Delete</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

## Anatomy

```
Dialog
  DialogTrigger (asChild = wraps a Button)
  DialogContent [showCloseButton]
    DialogHeader
      DialogTitle
      DialogDescription
    {your body}
    DialogFooter [showCloseButton]
      DialogClose
```

## Best practices

- **Always include `DialogTitle`.** Required for screen reader
  announcements. Use `className="sr-only"` if visually hidden.
- **Use `DialogDescription` for context.** A one-line explanation of
  the choice the user is about to make.
- **Reserve for blocking decisions.** Routine UI lives inline; Dialog
  is for moments that warrant interrupting the user.
- **Pair destructive actions with a clear `Cancel`.** The default
  close button alone is not enough for irreversible flows.
