`ContextMenu` is the right-click sibling of `DropdownMenu`. It anchors
to a region rather than a trigger button - wrap the region in
`<ContextMenuTrigger>`. Right-click is invisible to most users, so
treat ContextMenu as a power-user accelerator: every option here
should also be reachable through a button, DropdownMenu, or keyboard
shortcut elsewhere.

```tsx
import {
  ContextMenu,
  ContextMenuContent,
  ContextMenuItem,
  ContextMenuSeparator,
  ContextMenuShortcut,
  ContextMenuTrigger,
} from "@gremorie/rx-overlays";

export function CanvasArea() {
  return (
    <ContextMenu>
      <ContextMenuTrigger className="grid h-64 place-items-center rounded border border-dashed">
        Right-click here
      </ContextMenuTrigger>
      <ContextMenuContent>
        <ContextMenuItem>
          Cut <ContextMenuShortcut>Cmd+X</ContextMenuShortcut>
        </ContextMenuItem>
        <ContextMenuItem>
          Copy <ContextMenuShortcut>Cmd+C</ContextMenuShortcut>
        </ContextMenuItem>
        <ContextMenuItem>
          Paste <ContextMenuShortcut>Cmd+V</ContextMenuShortcut>
        </ContextMenuItem>
        <ContextMenuSeparator />
        <ContextMenuItem variant="destructive">Delete</ContextMenuItem>
      </ContextMenuContent>
    </ContextMenu>
  );
}
```

## Anatomy

```
ContextMenu
  ContextMenuTrigger (wraps the region)
  ContextMenuContent
    ContextMenuLabel
    ContextMenuItem [variant="destructive"]
    ContextMenuCheckboxItem
    ContextMenuRadioGroup
      ContextMenuRadioItem
    ContextMenuSeparator
    ContextMenuSub
      ContextMenuSubTrigger
      ContextMenuSubContent
    ContextMenuShortcut
```

## Best practices

- **Never ship as the only path.** Right-click is invisible to most
  users. Always provide a visible alternative.
- **Mirror DropdownMenu API.** Same items, same variants, same sub-menu
  shape - just a different trigger surface.
