`Alert` is a message anchored inside page flow - distinct from
`Sonner` (transient, floating) and `AlertDialog` (modal). Two
variants: `default` (neutral surface) and `destructive` (error tone).
Convey informational, success, and warning intent through a leading
icon rather than expanding variants - keeps the surface consistent
and the API minimal.

        <AlertTitle>Save failed</AlertTitle>

## Anatomy

```
Alert [variant="default" | "destructive"]
  <Icon />            (optional leading icon)
  AlertTitle
  AlertDescription
```

## Best practices

- **Use a leading icon to convey intent.** `Info`, `CheckCircle2`,
  `AlertTriangle`, `XCircle` from `lucide-react` are the conventional
  set.
- **Reserve `destructive` for errors.** Warnings get a yellow icon
  inside the default surface - they are informational, not failed
  state.
- **Pair with Sonner, not replace it.** Alert is for messages that
  should remain visible (form errors, system status); Sonner is for
  ephemeral feedback.
