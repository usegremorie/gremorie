`Tabs` is the canonical sibling-content switcher: profile sections,
settings categories, form steps. Built on Radix Tabs - full keyboard
navigation (arrow keys), focus management, and ARIA semantics.

```tsx
import {
  Tabs,
  TabsContent,
  TabsList,
  TabsTrigger,
} from "@gremorie/rx-navigation";

export function AccountTabs() {
  return (
    <Tabs defaultValue="profile">
      <TabsList>
        <TabsTrigger value="profile">Profile</TabsTrigger>
        <TabsTrigger value="billing">Billing</TabsTrigger>
        <TabsTrigger value="api">API keys</TabsTrigger>
      </TabsList>
      <TabsContent value="profile">Profile fields...</TabsContent>
      <TabsContent value="billing">Billing details...</TabsContent>
      <TabsContent value="api">API keys list...</TabsContent>
    </Tabs>
  );
}
```

</Tab>

## Anatomy

```
Tabs [defaultValue] [value] [onValueChange] [orientation]
  TabsList
    TabsTrigger [value]
  TabsContent [value]
```

## Best practices

- **Always render `TabsTrigger` inside `TabsList`.** Triggers placed
  outside the list lose keyboard navigation semantics.
- **Use 2-6 tabs.** Above that, consider a Sidebar or a Select.
- **Match label tone.** All single nouns ("Profile", "Billing") or all
  short verbs ("View", "Edit") - mixing types reads inconsistent.
