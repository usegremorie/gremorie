title: Area Chart
description: "Cumulative quantity over a continuous domain."
---

`AreaChart` plots one or more series as filled areas over a categorical
X axis. Pass tabular `data`, a JSON-serializable `config` mapping each
value field to a `{ label, color }`, and the `xKey` that names the
category field. Built on the shared chart engine (`chart-context`,
`chart-frame`, `axis`, `cartesian-grid`, `area`).

## Usage

<Tabs items={["Angular", "React"]}>

<Tab value="Angular">

```ts
import { Component } from "@angular/core";
import { AreaChart } from "@gremorie/ng-data";
import type { ChartConfig, Datum } from "@gremorie/ng-data";

@Component({
  selector: "sales-area",
  imports: [AreaChart],
  template: `
    <area-chart [data]="data" [config]="config" xKey="month" />
  `,
})
export class SalesArea {
  readonly data: Datum[] = [
    { month: "Jan", desktop: 186, mobile: 80 },
    { month: "Feb", desktop: 305, mobile: 200 },
    { month: "Mar", desktop: 237, mobile: 120 },
  ];
  readonly config: ChartConfig = {
    desktop: { label: "Desktop", color: "var(--chart-1)" },
    mobile: { label: "Mobile", color: "var(--chart-2)" },
  };
}
```

</Tab>

<Tab value="React">

```tsx
import { AreaChart } from "@gremorie/rx-data";
import type { ChartConfig, Datum } from "@gremorie/rx-data";

const data: Datum[] = [
  { month: "Jan", desktop: 186, mobile: 80 },
  { month: "Feb", desktop: 305, mobile: 200 },
  { month: "Mar", desktop: 237, mobile: 120 },
];

const config: ChartConfig = {
  desktop: { label: "Desktop", color: "var(--chart-1)" },
  mobile: { label: "Mobile", color: "var(--chart-2)" },
};

export function SalesArea() {
  return <AreaChart data={data} config={config} xKey="month" />;
}
```

</Tab>

## Anatomy

```
AreaChart (figure)
  ChartFrame (svg)
    CartesianGrid (horizontal gridlines)
    Area (one per series)
    XAxis (category ticks)
    YAxis (value ticks)
  Table (sr-only data table)
```

## Best practices

- **Tokens for color.** Use `var(--chart-1..5)` instead of hex so the
  chart inherits the theme.
- **Two to three series.** Beyond that, prefer a line chart - overlapping
  filled areas become hard to read.
- **Provide a sr-only table.** The component already renders one from
  `data` and `config`; do not hide it.
