## Overview

`PieChart` is the styled wrapper of the headless charting engine for **parts of a whole**. Pass tabular `data`, the `nameKey` (the slice label field), and the `valueKey` (the numeric value field). Slice colors cycle through the `--chart-1` through `--chart-5` tokens automatically. Pass `donut` for a donut variant with a centered hole.

Reach for PieChart when **proportions matter** and you have **two to six categories** that sum to a meaningful total - traffic share by browser, revenue by tier, allocations by team. For seven or more categories, a [`BarChart`](/components/data/bar-chart) reads faster; for time-series proportions, a stacked [`AreaChart`](/components/data/area-chart) carries more information.

## API

### `<PieChart>`

The styled wrapper. Renders a `<figure role="img">` containing the chart `<svg>` with one `<path>` per slice, a wrapping legend of slice swatches plus labels, and an `sr-only` data table for assistive tech.

| Prop | Type | Default | Description |
|---|---|---|---|
| `data` | `readonly Datum[]` | - | Tabular data. One row per slice. Must include a string field at `nameKey` and a numeric field at `valueKey`. |
| `nameKey` | `string` | - | Name of the slice label field. Drives the legend and the `sr-only` table headers. |
| `valueKey` | `string` | - | Name of the numeric value field. Slices are sized proportionally to its sum. |
| `donut` | `boolean` | `false` | When `true`, carves out a centered hole, producing a donut chart with `innerRadiusRatio` of 0.6. |
| `className` | `string` | - | Merged onto the outer figure. The bundled style supplies a rounded card surface plus a square aspect. |

### Headless primitives exposed

| Export | Role |
|---|---|
| `ChartFrame` | Root `<svg>`, owns size via `ResizeObserver`, provides the polar projection context. |
| `Pie` | Render-prop primitive yielding `{ slices }`. Each slice carries `{ d, name, index, value, fraction }` so you draw the `<path>` and pick the color yourself. |

### Types

| Type | Shape |
|---|---|
| `Datum` | `Record<string, string \| number>` - one row of chart data. |

<Callout type="info">
`PieChart` does **not** take a `config` prop - slice colors are auto-assigned from the `--chart-1` through `--chart-5` tokens in row order. For explicit per-slice colors, drop down to the headless `<Pie>` and supply your own `fill`.
</Callout>

## Composition

1. **Pick `nameKey` and `valueKey`** - the label column and the numeric column. Both must exist on every row.
2. **Order rows by magnitude** before passing them in. The biggest slice should sit at the 12-o'clock anchor so the eye reads it first.
3. **Toggle `donut`** when you want to embed a total or label at the center, or when the surrounding card is large enough that a solid pie feels too heavy.
4. **For more than five colors**, drop down to `<Pie>` and supply a longer color list. The bundled wrapper cycles `--chart-1` through `--chart-5`.

The chart auto-sizes to its parent via `ResizeObserver`. The figure forces `aspect-square` so the slices stay circular.

### Headless customization

```tsx
import { ChartFrame, Pie } from "@gremorie/rx-data";

export function ExplicitColorPie({ data, nameKey, valueKey, colorByName }) {
  return (
    <figure role="img" aria-label="Custom pie">
      <ChartFrame data={data} xKey={nameKey} className="mx-auto aspect-square max-h-[260px] w-full">
        <Pie valueKey={valueKey}>
          {({ slices }) =>
            slices.map((s) => (
              <path
                key={s.name}
                d={s.d}
                fill={colorByName[s.name]}
                stroke="var(--background)"
                strokeWidth={2}
              />
            ))
          }
        </Pie>
      </ChartFrame>
    </figure>
  );
}
```

Drop down to headless when you need explicit colors keyed by slice name, gradient fills, or a custom legend layout.

## Accessibility

- **Figure role**: the wrapper renders `<figure role="img">` with an auto-generated `aria-label` (`"Pie chart of visitors by browser"` or `"Donut chart of visitors by browser"`).
- **Mirror table**: the chart appends a visually hidden `<table>` listing every slice name and value. Screen readers navigate the data via native table commands.
- **Visible legend**: a wrapping `<ul>` of swatch plus label sits below the chart, providing the color-to-name mapping for sighted users without depending on tooltips.
- **Token contrast**: `--chart-1` through `--chart-5` ship with WCAG AA contrast against the card background in both light and dark themes.
- **Slice separation**: slices are stroked with the background color at 2 px, so adjacent slices stay distinguishable for users with color-vision deficiencies even when the colors are close.
- **Limit slice count**: above six slices, a pie becomes hard to read even with the legend. Reach for [`BarChart`](/components/data/bar-chart) instead.
