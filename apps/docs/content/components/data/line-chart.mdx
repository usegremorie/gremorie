## Overview

`LineChart` is the styled wrapper of the headless charting engine for **trend lines over an ordered category**. Pass tabular `data`, a serializable `config` that maps each value field to a `label` plus a `color`, and the `xKey` that names the category field. The chart wires up scales, the cartesian grid, axis ticks, and per-series stroked paths automatically, and ships an `sr-only` HTML table that mirrors the data for assistive tech.

Reach for LineChart when **shape and direction** are the headline - month-over-month conversion, weekly active users, deploys per day. Use [`AreaChart`](/components/data/area-chart) when the filled magnitude is the story; use [`BarChart`](/components/data/bar-chart) for discrete category comparisons; use [`ScatterChart`](/components/data/scatter-chart) when the X axis is itself numeric and continuous.

## API

### `<LineChart>`

The styled wrapper. Renders a `<figure role="img">` containing the chart `<svg>`, a cartesian grid, X and Y axes, one stroked `<Line>` per series in `config`, and an `sr-only` data table for assistive tech.

| Prop | Type | Default | Description |
|---|---|---|---|
| `data` | `readonly Datum[]` | - | Tabular data. Each row maps string keys to numeric or category values. The column at `xKey` is the category; every key in `config` must exist on every row as a numeric value. |
| `config` | `ChartConfig` | - | Maps each series key to `{ label, color }`. The `color` is any CSS color or token, typically `"var(--chart-1)"` through `"var(--chart-5)"`. |
| `xKey` | `string` | - | Name of the category field on each `Datum`. Drives the X axis ticks. |
| `className` | `string` | - | Merged onto the outer figure. Use for sizing escape hatches; the bundled style already supplies a rounded card surface plus aspect ratio. |

### Headless primitives exposed

| Export | Role |
|---|---|
| `ChartFrame` | Root `<svg>`, owns size via `ResizeObserver`, provides scales and shared Y domain. |
| `CartesianGrid` | Render-prop primitive yielding horizontal grid lines for the current Y ticks. |
| `XAxis` / `YAxis` | Render-prop primitives yielding tick positions and labels. |
| `Line` | Per-series stroked `<path>`. Self-registers with the frame so the shared Y domain covers it. |

### Types

| Type | Shape |
|---|---|
| `Datum` | `Record<string, string \| number>` - one row of chart data. |
| `ChartConfig` | `Record<string, { label: string; color: string; format?: string }>` - JSON-serializable series map. |

## Composition

1. **Pick `xKey`** first - the ordered category that runs along the bottom.
2. **Author `config`** mapping each value field to a label and a token-driven color. Color tokens (`--chart-1` through `--chart-5`) already encode legible contrast across light and dark themes.
3. **Pass `data`** as an array of records. The headless engine recomputes scales on every render, so stale rows are safe.
4. **For custom layouts** (different stroke widths, log Y axis, smoothed curves), drop down to `<ChartFrame>` plus `<Line>` and supply your own children.

The chart auto-sizes to its parent via `ResizeObserver`, so the wrapping surface controls the width. The default class on the figure pins the chart to `aspect-video` for a stable card shape.

### Headless customization

```tsx
import {
  ChartFrame,
  CartesianGrid,
  XAxis,
  YAxis,
  Line,
} from "@gremorie/rx-data";

export function ThickLineChart({ data, config, xKey }) {
  return (
    <figure role="img" aria-label="Custom line chart">
      <ChartFrame data={data} xKey={xKey} className="aspect-[2/1] w-full text-muted-foreground">
        <CartesianGrid>{(lines) => /* custom grid */ null}</CartesianGrid>
        {Object.entries(config).map(([key, entry]) => (
          <Line
            key={key}
            dataKey={key}
            color={entry.color}
            strokeWidth={3}
          />
        ))}
        <XAxis>{({ ticks, labelY }) => /* custom ticks */ null}</XAxis>
        <YAxis>{({ ticks, labelX }) => /* custom ticks */ null}</YAxis>
      </ChartFrame>
    </figure>
  );
}
```

Drop down to headless when you need thicker strokes, dashed segments, dot markers, or a log axis. The render-prop primitives hand you the projected positions so the engine still does the math.

## Accessibility

- **Figure role**: the wrapper renders `<figure role="img">` with an auto-generated `aria-label` (`"Line chart of Desktop, Mobile by month"`).
- **Mirror table**: the chart appends a visually hidden `<table>` carrying the same rows, columns, and labels. Screen readers can navigate the data via native table commands.
- **Token contrast**: `--chart-1` through `--chart-5` are tuned for WCAG AA contrast against the card background in both light and dark themes.
- **Static rendering**: no animation by default, so users with `prefers-reduced-motion: reduce` are not penalized.
- **No streaming UI**: charts do not implement a streaming append affordance. For live data, throttle parent updates or render inside a `useDeferredValue` boundary.
