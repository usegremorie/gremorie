`Badge` is the small, all-caps label that lives next to titles, on
top of cards, and inside lists - status tags, counts, categories.
Variants: `default`, `secondary`, `destructive`, `outline`. Supports
`asChild` via Radix Slot for composition with anchors or buttons.

## Variants

- `default` - solid, primary tone. Use for high-signal status.
- `secondary` - softer surface. Use for neutral status.
- `destructive` - red tone. Reserve for errors and dangerous state.
- `outline` - outlined only. Use for less prominent metadata.

## Best practices

- **Keep the text short.** One or two words. If you need more, use a
  Card or a list cell.
- **Avoid `destructive` for non-errors.** It carries a hazard
  semantic - using it for "Premium" is a tone mismatch.
- **Compose with `asChild` for interactive badges.** Wrap an anchor or
  button to inherit Badge styling while keeping link/button semantics.
