`Collapsible` is the minimal building block for "show more" toggles -
a card's "more details", a sidebar group's expand / collapse, an
inline disclosure in long copy. Three parts: `Collapsible` (root),
`CollapsibleTrigger`, `CollapsibleContent`. For multiple coordinated
sections, use `Accordion` instead.

```tsx

import {
  Collapsible,
  CollapsibleContent,
  CollapsibleTrigger,
} from "@gremorie/rx-display";

export function MoreOptions() {
  return (
    <Collapsible>
      <CollapsibleTrigger asChild>
        <Button variant="ghost">Advanced settings</Button>
      </CollapsibleTrigger>
      <CollapsibleContent className="mt-2 grid gap-2">
        <p>Hidden details appear here.</p>
      </CollapsibleContent>
    </Collapsible>
  );
}
```

## Anatomy

```
Collapsible [open] [defaultOpen]
  CollapsibleTrigger (asChild)
  CollapsibleContent
```

## Best practices

- **Default closed.** A Collapsible that defaults open defeats its
  purpose - if it should always be visible, just render the content.
- **Reach for Accordion when sections coordinate.** Collapsible is a
  single switch; Accordion enforces "one at a time" or "many open"
  semantics.
