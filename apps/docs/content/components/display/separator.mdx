`Separator` is the styled `<hr>` of the design system. Use sparingly -
prefer spacing for visual grouping. Reach for `Separator` only when a
hard line carries semantic meaning: between logically distinct groups
of items, between a menu's commands and shortcuts, between a card's
content and its footer.

## Best practices

- **Spacing first, separator second.** A clear gap usually does the job
  without ink. Add a Separator only when grouping is ambiguous.
- **Use `orientation="vertical"` for inline groups.** Common in nav
  bars, breadcrumbs, and toolbar clusters.
