`Input` is the canonical single-line text field. It accepts every native
HTML `type` (`text`, `email`, `password`, `number`, `search`, `tel`,
`url`, `file`...) and renders the same token-driven visual states:
default border, `focus-visible` ring, and `aria-invalid` destructive
border.

## States

| State            | Trigger                              | Visual                          |
| ---------------- | ------------------------------------ | ------------------------------- |
| Default          | -                                    | `border-input` border.          |
| Focus            | `focus-visible`                      | `ring-ring/50` ring + border.   |
| Invalid          | `aria-invalid="true"`                | `border-destructive` border.    |
| Disabled         | `disabled`                           | 50% opacity, no pointer events. |

## Best practices

- **Always pair with `Label`.** Use `htmlFor` matching the input `id`.
- **Set `aria-invalid` via your form library.** The styling responds
  automatically. Don't apply `border-red-500` manually.
- **Use the native `type`.** `type="email"` triggers the right keyboard
  on mobile and primes browser validation.
