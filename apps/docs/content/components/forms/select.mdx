title: Select
description: Dropdown chooser for short fixed lists - built on Radix Select.
---

`Select` is the canonical dropdown picker. Use it when you have a fixed
list of 5-30 options and the user should not need to search. Beyond
that, reach for a Combobox.

The visible trigger and value live in the page flow; the listbox is
portalled and animated by Radix.

```tsx
import {
  Select,
  SelectContent,
  SelectGroup,
  SelectItem,
  SelectLabel,
  SelectSeparator,
  SelectTrigger,
  SelectValue,
} from "@gremorie/rx-forms";

export function ModelPicker() {
  return (
    <Select defaultValue="gpt-4o">
      <SelectTrigger className="w-48">
        <SelectValue placeholder="Model" />
      </SelectTrigger>
      <SelectContent>
        <SelectGroup>
          <SelectLabel>OpenAI</SelectLabel>
          <SelectItem value="gpt-4o">GPT-4o</SelectItem>
          <SelectItem value="gpt-4o-mini">GPT-4o mini</SelectItem>
        </SelectGroup>
        <SelectSeparator />
        <SelectGroup>
          <SelectLabel>Anthropic</SelectLabel>
          <SelectItem value="claude-4.5">Claude 4.5</SelectItem>
        </SelectGroup>
      </SelectContent>
    </Select>
  );
}
```

## Best practices

- **Always nest items in `SelectGroup`.** Loose `SelectItem` instances
  break grouping for screen readers.
- **Use Combobox beyond ~30 options.** Select is for fixed short lists.
- **For actions, use `DropdownMenu` instead.** Select holds values
  (USA, Brazil, Japan); DropdownMenu holds verbs (Edit, Delete).
