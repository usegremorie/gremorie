`Label` is the static label sitting above any input control. When
`htmlFor` matches a control's `id`, clicking the label focuses (or
activates) the control - it doubles as a click target.

The default styling dims the label automatically when the related input
is disabled, via `peer-disabled` and `group-data-[disabled=true]`
selectors.

## Best practices

- **Always set `htmlFor`.** Without it the label is non-functional and
  screen readers cannot announce the associated control.
- **Don't replace with raw `<span>` or `<p>`.** Form controls need the
  accessibility wiring Radix Label provides.
