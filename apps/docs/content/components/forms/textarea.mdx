title: Textarea

---

`Textarea` is the multi-line companion to `Input`. It uses the
modern `field-sizing: content` CSS property to grow with its content
automatically - no manual `rows` count and no JS measurement.

For very long-form text (rich text, markdown editor, code), prefer a
dedicated editor over `Textarea`. For composers with toolbars or
inline addons, compose with `InputGroup` + `InputGroupTextarea`.

## States

| State    | Trigger              | Visual                            |
| -------- | -------------------- | --------------------------------- |
| Default  | -                    | `border-input` border.            |
| Focus    | `focus-visible`      | `ring-ring/50` ring + border.     |
| Invalid  | `aria-invalid="true"`| `border-destructive` border.      |
| Disabled | `disabled`           | 50% opacity, no pointer events.   |

## Best practices

- **Always pair with `Label`.** Use `htmlFor` matching the textarea `id`.
- **Skip `rows`.** Auto-sizing is enabled by default; specify
  `min-h-*` / `max-h-*` Tailwind classes if you want hard bounds.
- **For composer toolbars (Submit, Attachments, Send), wrap in
  `InputGroup`.** That gives you the multi-line `InputGroupTextarea`
  variant with addon slots.
