`Queue` is part of the **AI chatbot family** brought from Bridge into Gremorie in Fase 5j. It composes on top of other Gremorie primitives (`@gremorie/rx-display`, `@gremorie/rx-overlays`, `@gremorie/rx-forms`, ...) and ships with full Bridge parity.

```tsx
import {
  Queue,
  QueueTrigger,
  QueueContent,
  QueueMessage,
  QueueMessageText,
  QueueMessageAttachment,
} from '@gremorie/rx-ai';

export function Demo() {
  return (
    <Queue>
      <QueueTrigger count={queued.length} />
      <QueueContent>
        {queued.map((m) => (
          <QueueMessage key={m.id}>
            <QueueMessageText>{getText(m)}</QueueMessageText>
          </QueueMessage>
        ))}
      </QueueContent>
    </Queue>
  );
}
```

## Notes

- This primitive was migrated from Bridge into `@gremorie/rx-ai` as part of the Fase 5j chatbot batch (Bridge parity for the AI category).
- All cross-Gremorie imports use the published `@gremorie/rx-*` packages; the registry mirrors that into per-primitive source so consumers can `npx gremorie add rx-queue` without pulling unrelated AI components.
