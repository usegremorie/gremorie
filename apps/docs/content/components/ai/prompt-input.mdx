---
title: PromptInput
description: Rich prompt input - textarea, toolbar, submit with lifecycle states (ready / submitted / streaming / stop / error).
---

`PromptInput` is the **input area** of an AI chat. It auto-resizes the
textarea, exposes a toolbar slot for actions (attach, mic, web search,
model select), and renders a Submit button whose icon flips by status -
`ready` shows an arrow, `submitted` shows a spinner, `streaming` shows
a stop control.

Both editions ship the full family today:

- **Angular**: `@gremorie/ng-ai` - full family including action menus,
  model selector, attachment hover-card previews.
- **React**: `@gremorie/rx-ai` - full Bridge parity. Composes on top
  of `InputGroup` + `InputGroupButton` + `InputGroupTextarea` from
  `@gremorie/rx-forms`, so the textarea + addon toolbar render as one
  cohesive surface. Includes ActionMenu (DropdownMenu), ModelSelector
  (Select), attachment HoverCard previews, Speech button (Web Speech
  API), Command palette wrappers, the Cmd+K floating palette via
  `PromptInputCommandDialog`, and lightweight Tabs panels
  (`PromptInputTab*`) for grouping options inside menus. Pulls in
  `rx-button`, `rx-input-group`, `rx-select`, `rx-dialog`,
  `rx-dropdown-menu`, `rx-hover-card`, and `rx-command` as registry
  dependencies.

## Usage

<Tabs items={["Angular", "React"]}>

<Tab value="Angular">

```ts
import {
  PromptInput,
  PromptInputSubmit,
  PromptInputTextarea,
  PromptInputToolbar,
} from '@gremorie/ng-ai';

@Component({
  selector: 'app-chat-box',
  standalone: true,
  imports: [PromptInput, PromptInputTextarea, PromptInputToolbar, PromptInputSubmit],
  template: `
    <gremorie-prompt-input [(value)]="draft" (gremoriePromptSubmit)="onSubmit($event)">
      <gremorie-prompt-input-textarea placeholder="Ask anything"></gremorie-prompt-input-textarea>
      <gremorie-prompt-input-toolbar>
        <gremorie-prompt-input-submit></gremorie-prompt-input-submit>
      </gremorie-prompt-input-toolbar>
    </gremorie-prompt-input>
  `,
})
export class ChatBoxComponent {
  draft = signal('');
  onSubmit(event: PromptInputSubmitEvent) {
    sendMessage(event.value, event.attachments);
  }
}
```

## Anatomy

```
PromptInput (root form, owns state machine)
  PromptInputBody
    PromptInputAttachments (lists attached files via render-prop)
      PromptInputAttachment (one chip + HoverCard preview)
    PromptInputHeader (optional, model selector slot)
    PromptInputTextarea (auto-resize, Enter submits, Shift+Enter newline)
    PromptInputFooter
      PromptInputTools (left, action buttons)
        PromptInputActionMenu (dropdown with attachments / actions)
        PromptInputSpeechButton (Web Speech API toggle)
        PromptInputButton (web search, custom actions)
        PromptInputSelect (model selector)
      PromptInputSubmit (right, status-driven)
```

## InputGroup composition (React)

The React edition wraps `<form>` children in an `InputGroup` and
implements `PromptInputButton`, `PromptInputSubmit`, and
`PromptInputTextarea` on top of `InputGroupButton` /
`InputGroupTextarea`. That gives you:

- A single rounded surface with shared focus / invalid / disabled
  states across the textarea and its addons.
- Block-aligned header / footer toolbars via `InputGroupAddon`
  underneath the hood, so the composer reads as one unit instead of
  three stacked rectangles.
- Sizing variants on every toolbar button (`xs`, `sm`, `icon-xs`,
  `icon-sm`) that line up with the group height automatically.

`PromptInputButton` auto-picks `sm` when the button has both an icon
and a label, and `icon-sm` when it's icon-only. Override `size` to
pin it.

## Command palette (React)

`PromptInputCommandDialog` exposes the canonical Cmd+K floating
palette wrapping the same `Command` primitive the inline picker
uses. Use it for quick-action launchers, model switchers, or
chat-history search:

```tsx
import { useEffect, useState } from "react";
import {
  PromptInputCommand,
  PromptInputCommandDialog,
  PromptInputCommandEmpty,
  PromptInputCommandGroup,
  PromptInputCommandInput,
  PromptInputCommandItem,
  PromptInputCommandList,
} from "@gremorie/rx-ai";

export function QuickActionsPalette() {
  const [open, setOpen] = useState(false);

  useEffect(() => {
    const handler = (e: KeyboardEvent) => {
      if (e.key === "k" && (e.metaKey || e.ctrlKey)) {
        e.preventDefault();
        setOpen((v) => !v);
      }
    };
    document.addEventListener("keydown", handler);
    return () => document.removeEventListener("keydown", handler);
  }, []);

  return (
    <PromptInputCommandDialog open={open} onOpenChange={setOpen}>
      <PromptInputCommandInput placeholder="Type a command..." />
      <PromptInputCommandList>
        <PromptInputCommandEmpty>No results.</PromptInputCommandEmpty>
        <PromptInputCommandGroup heading="Actions">
          <PromptInputCommandItem>New chat</PromptInputCommandItem>
          <PromptInputCommandItem>Clear history</PromptInputCommandItem>
        </PromptInputCommandGroup>
      </PromptInputCommandList>
    </PromptInputCommandDialog>
  );
}
```

## States

`status` drives the Submit button icon and behaviour.

| Status        | Icon              | Behaviour                                                       |
| ------------- | ----------------- | --------------------------------------------------------------- |
| `ready`       | Arrow up          | Submit normally on Enter or click. Disabled if textarea empty.  |
| `submitted`   | Spinner           | Awaiting response. Submit disabled, prevents double submission. |
| `streaming`   | Stop (square)     | Allow the user to interrupt generation.                         |
| `error`       | X                 | Indicates an error - host can retry on click.                   |

## When to use

- The input area in any AI chat surface.
- A "compose" surface with upload, voice input, and web-search shortcuts.

## When not to use

- A plain text input - use a native textarea or your design system's
  Textarea primitive.
- A multi-field form (signup, settings) - use Form.

## Best practices

- **Always pass `status`** to Submit. Without it, the icon stays on
  `ready` even while a response is streaming.
- **Disable the textarea** while `submitted` or `streaming`. Both
  editions support this via the textarea `disabled` prop.
- **Use a meaningful placeholder.** "Ask anything..." beats "Message".
- **Constrain attachments** with `accept` and `maxFileSize` - never
  trust the client.
- **Pass `textareaRef`** to `PromptInputSpeechButton` so dictation
  appends to the live textarea value.

## Accessibility

- Both editions render a real `<form>` element. Enter submits, Shift+Enter
  inserts a newline. Backspace on an empty textarea removes the last
  attachment.
- Submit carries `aria-label` based on `status` ("Submit message",
  "Stop generation").
- Speech button uses the Web Speech API and degrades gracefully when the
  browser does not support it - the button is disabled instead of hidden,
  with `aria-label="Toggle voice input"`.
- Drag-drop zones are focusable and announce file additions to assistive
  tech via standard `<input type="file">` semantics.
