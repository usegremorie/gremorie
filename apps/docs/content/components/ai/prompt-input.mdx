---
title: PromptInput
description: Rich prompt input - textarea, toolbar, submit with lifecycle states (ready / submitted / streaming / stop / error).
---

`PromptInput` is the **input area** of an AI chat. It auto-resizes the
textarea, exposes a toolbar slot for actions (attach, mic, web search,
model select), and renders a Submit button whose icon flips by status -
`ready` shows an arrow, `submitted` shows a spinner, `streaming` shows
a stop control.

Two editions ship today:

- **Angular**: `@gremorie/ng-ai` - full family including action menus,
  model selector, attachment hover-card previews.
- **React**: `@gremorie/rx-ai` (pilot) - state machine, drag-drop,
  paste-to-attach, status-driven Submit. Action menus, hover-card
  previews, model selector, and command palette land in the next
  iteration once rx-forms and rx-overlays ship.

## Install

<Tabs items={["Angular", "React"]}>

<Tab value="Angular">

```bash
npx gremorie add ng-prompt-input
```

This pulls in `ng-utils` (the `cn()` helper) and `ng-attachments` (the Attachment family used by drag-drop and paste-to-attach) as registry dependencies. If you only need the attachment UI in isolation, install it directly:

```bash
npx gremorie add ng-attachments
```


Pulls in `rx-utils` (the `cn()` helper). The React edition is a single-file pilot - attachments, action menu, model select, and hover-card previews land in a later iteration.

## Usage

<Tabs items={["Angular", "React"]}>

<Tab value="Angular">

```ts
import {
  PromptInput,
  PromptInputSubmit,
  PromptInputTextarea,
  PromptInputToolbar,
} from '@gremorie/ng-ai';

@Component({
  selector: 'app-chat-box',
  standalone: true,
  imports: [PromptInput, PromptInputTextarea, PromptInputToolbar, PromptInputSubmit],
  template: `
    <gremorie-prompt-input [(value)]="draft" (gremoriePromptSubmit)="onSubmit($event)">
      <gremorie-prompt-input-textarea placeholder="Ask anything"></gremorie-prompt-input-textarea>
      <gremorie-prompt-input-toolbar>
        <gremorie-prompt-input-submit></gremorie-prompt-input-submit>
      </gremorie-prompt-input-toolbar>
    </gremorie-prompt-input>
  `,
})
export class ChatBoxComponent {
  draft = signal('');
  onSubmit(event: PromptInputSubmitEvent) {
    sendMessage(event.value, event.attachments);
  }
}
```

</Tab>

<Tab value="React">

```tsx
"use client";

import { useState } from "react";
import {
  PromptInput,
  PromptInputBody,
  PromptInputFooter,
  PromptInputSubmit,
  PromptInputTextarea,
  type ChatStatus,
} from "@gremorie/rx-ai";

export function ChatBox() {
  const [status, setStatus] = useState<ChatStatus>("ready");

  return (
    <PromptInput
      onSubmit={(message) => sendMessage(message.text, message.files)}
    >
      <PromptInputBody>
        <PromptInputTextarea placeholder="Ask anything" />
        <PromptInputFooter>
          <span />
          <PromptInputSubmit status={status} />
        </PromptInputFooter>
      </PromptInputBody>
    </PromptInput>
  );
}
```

## Anatomy

```
PromptInput (root form, owns state machine)
  PromptInputBody
    PromptInputAttachments (optional, lists attached files)
    PromptInputHeader (optional, model selector slot)
    PromptInputTextarea (auto-resize, Enter submits, Shift+Enter newline)
    PromptInputFooter
      PromptInputTools (left, action buttons)
        PromptInputButton (attach, mic, web search, ...)
      PromptInputSubmit (right, status-driven)
```

## States

`status` drives the Submit button icon and behaviour.

| Status        | Icon              | Behaviour                                                       |
| ------------- | ----------------- | --------------------------------------------------------------- |
| `ready`       | Arrow up          | Submit normally on Enter or click. Disabled if textarea empty.  |
| `submitted`   | Spinner           | Awaiting response. Submit disabled, prevents double submission. |
| `streaming`   | Stop (square)     | Allow the user to interrupt generation.                         |
| `error`       | X                 | Indicates an error - host can retry on click.                   |

## Drag-drop

Both editions accept files dragged onto the form. Pass `globalDrop` to
accept drops anywhere on the document.

## When to use

- The input area in any AI chat surface.
- A "compose" surface with upload, voice input, and web-search shortcuts.

## When not to use

- A plain text input - use a native textarea or your design system's
  Textarea primitive.
- A multi-field form (signup, settings) - use Form.

## Best practices

- **Always pass `status`** to Submit. Without it, the icon stays on
  `ready` even while a response is streaming.
- **Disable the textarea** while `submitted` or `streaming`. Both
  editions support this via the textarea `disabled` prop.
- **Use a meaningful placeholder.** "Ask anything..." beats "Message".
- **Constrain attachments** with `accept` and `maxFileSize` - never
  trust the client.

## Accessibility

- Both editions render a real `<form>` element. Enter submits, Shift+Enter
  inserts a newline. Backspace on an empty textarea removes the last
  attachment.
- Submit carries `aria-label` based on `status` ("Submit message",
  "Stop generation").
- Drag-drop zones are focusable and announce file additions to assistive
  tech via standard `<input type="file">` semantics.
