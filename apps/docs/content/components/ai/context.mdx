title: Context

`Context` is part of the **AI chatbot family** brought from Bridge into Gremorie in Fase 5j. It composes on top of other Gremorie primitives (`@gremorie/rx-display`, `@gremorie/rx-overlays`, `@gremorie/rx-forms`, ...) and ships with full Bridge parity.

## Install

<Tabs items={["Angular", "React"]}>

<Tab value="Angular">

Em breve. A categoria AI no Angular ainda so cobre PromptInput. Use a edicao React por enquanto, ou abra uma issue para priorizar este primitivo.

</Tab>

<Tab value="React">

```bash
npx gremorie add rx-context
```

Pulls in the registry dependencies declared for `rx-context` (cross-primitive Gremorie components - see `registry.json`).

</Tab>

</Tabs>

```tsx
import {
  Context,
  ContextTrigger,
  ContextContent,
  ContextContentHeader,
  ContextContentBody,
  ContextContentFooter,
  ContextInputUsage,
  ContextOutputUsage,
  ContextReasoningUsage,
  ContextCacheUsage,
  ContextCacheCreationUsage,
} from '@gremorie/rx-ai';

export function Demo() {
  return (
    <Context usage={usage} modelId="anthropic/claude-sonnet-4">
      <ContextTrigger />
      <ContextContent>
        <ContextContentBody>
          <ContextInputUsage />
          <ContextOutputUsage />
        </ContextContentBody>

  );
}
```

## Notes

- This primitive was migrated from Bridge into `@gremorie/rx-ai` as part of the Fase 5j chatbot batch (Bridge parity for the AI category).
- All cross-Gremorie imports use the published `@gremorie/rx-*` packages; the registry mirrors that into per-primitive source so consumers can `npx gremorie add rx-context` without pulling unrelated AI components.
