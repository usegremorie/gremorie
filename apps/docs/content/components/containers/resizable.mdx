`Resizable` composes three pieces on top of `react-resizable-panels`:
`ResizablePanelGroup` is the frame, `ResizablePanel` claims a flex
region, `ResizableHandle` is the drag-and-drop divider between two
panels. Pass `withHandle` to get a visible grip.

Use Resizable for **dev tools and pro apps** - code editors, email
clients, file managers. Consumer-facing surfaces almost always read
better with fixed proportions plus responsive breakpoints.

  ResizableHandle,

  ResizablePanelGroup,

  return (
    <ResizablePanelGroup
      direction="horizontal"
      className="min-h-[300px] rounded-lg border"
    >
      <ResizablePanel defaultSize={25}>
        <div className="flex h-full items-center justify-center p-6">
          Sidebar
        </div>
      </ResizablePanel>
      <ResizableHandle withHandle />
      <ResizablePanel defaultSize={75}>
        <div className="flex h-full items-center justify-center p-6">
          Content
        </div>
      </ResizablePanel>
    </ResizablePanelGroup>
  );
}

## Anatomy

```
ResizablePanelGroup [direction="horizontal" | "vertical"]
  ResizablePanel [defaultSize, minSize, maxSize]
  ResizableHandle [withHandle?]
  ResizablePanel
```

## Best practices

- **Pick a direction up front.** Horizontal panels are the typical
  pattern (sidebar + main). Vertical panels suit terminal stacks or
  preview-on-top, editor-below editors.
- **Show the handle when the resize is a primary affordance.** Use
  `withHandle` on tools where users frequently rearrange the layout;
  leave it off when the dragger is a power-user detail.
- **Persist sizes for serious tools.** `react-resizable-panels` exposes
  `onLayout` and `id` to round-trip sizes through localStorage.
- **Default to fixed layouts on consumer surfaces.** Drag-to-resize is
  invisible to most users; lean on responsive breakpoints instead.
