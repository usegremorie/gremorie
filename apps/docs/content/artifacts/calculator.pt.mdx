## Visão geral

Um artifact interativo para tool calls que precisam de uma UI rápida. O modelo
emite `{ a, b, op }`; o artifact renderiza inputs editáveis, um toggle de
operador e um resultado ao vivo. O usuário pode ajustar os valores sem fazer
um round-trip de volta ao modelo.

Este artifact demonstra o extremo stateful do espectro — a maioria dos
artifacts renderiza uma vez, mas widgets interativos mantêm seu próprio estado
depois que o payload inicial chega.

## Schema

O LLM retorna output estruturado que segue este shape:

```ts
{
  type: "calculator",
  a: number,
  b: number,
  op: "+" | "-" | "*" | "/",
}
```

## Anatomia

```text
Card
├─ CardHeader            CardTitle + type Badge
├─ number input pair     Label + Input[type=number] (A, B)
├─ operator toggle       Button per operator (aria-pressed)
├─ result row            muted strip with the computed value
└─ reset                 ghost Button restoring defaults
```

'use client';

import { useState } from 'react';
import {
  Badge,
  Card,

  CardHeader,
  CardTitle,

import { Button, Input, Label } from '@gremorie/rx-forms';
import { RotateCcw } from 'lucide-react';

type Op = '+' | '-' | '*' | '/';

function compute(a: number, b: number, op: Op): number {
  switch (op) {
    case '+':
      return a + b;
    case '-':
      return a - b;
    case '*':
      return a * b;
    case '/':
      return b === 0 ? NaN : a / b;
  }
}

  const [a, setA] = useState(48);

            <CardTitle>Quick calculator</CardTitle>
            <CardDescription>
              Interactive widget generated from a tool call.
            </CardDescription>
          </div>
          <Badge variant="outline">Tool widget</Badge>
        </div>
      </CardHeader>
      <CardContent className="flex flex-col gap-4">
        {/* Inputs, operator toggle, result, reset... */}
      </CardContent>

}

## Comportamento de streaming

<Callout type="info">
  Este artifact renderiza depois que o payload é parseado, e então assume seu
  próprio estado. O modelo semeia os valores iniciais; o usuário conduz tudo que
  vem depois. Tool calls subsequentes podem re-semear.
</Callout>
