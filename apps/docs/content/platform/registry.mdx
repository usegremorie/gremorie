The Gremorie registry is the source of truth for every primitive,
block, and artifact. It is a flat directory of JSON files served from
this site, indexed by [`/r/registry.json`](/r/registry.json).

## The model

Gremorie follows the same shape pioneered by `shadcn/ui`: you do not
install a versioned package. You copy the source into your project, and
the code becomes yours.

The benefits compound:

- No black-box upgrade path -- you control every file
- LLMs can read the registry directly and emit valid usage
- The same source ships to Angular and React via separate registries

## Layout

| Path                                                       | Purpose                                       |
| ---------------------------------------------------------- | --------------------------------------------- |
| [`/r/registry.json`](/r/registry.json)                     | Index of every item (id, framework, category) |
| [`/r/<framework>/<item>.json`](/r/rx/rx-button.json)       | Full item: source files, deps, registry deps  |
| [`/schema/registry-item.json`](/schema/registry-item.json) | JSON Schema for a single item                 |

## Item shape

Each item is a single JSON document with embedded source files:

```json
{
  "name": "rx-button",
  "type": "registry:block",
  "framework": "rx",
  "title": "Button (React)",
  "categories": ["core", "primitives", "forms"],
  "dependencies": ["class-variance-authority", "radix-ui"],
  "registryDependencies": ["rx-utils"],
  "files": [
    {
      "path": "src/lib/button/button.tsx",
      "type": "registry:component",
      "target": "src/components/gremorie/forms/button/button.tsx",
      "content": "// component source here"
    }
  ]
}
```

- **`dependencies`** -- npm packages the consumer must install
- **`registryDependencies`** -- other registry items the CLI must pull in
- **`files`** -- embedded source written verbatim to the project; each
  file's `target` is where the CLI writes it in your repo

## Consuming the registry yourself

The CLI is the recommended way. But the registry is a flat HTTP API; any
tool can read it.

```bash
# Get the index
curl https://gremorie.com/r/registry.json | jq '.items[0:5]'

# Fetch a single item
curl https://gremorie.com/r/rx/rx-button.json | jq '.files[0].content'
```

This is the same path an MCP client takes when an LLM queries the
registry through the [MCP server](/platform/mcp).

## Editions

Two editions are published from the same monorepo:

- **`/r/rx/*`** -- React primitives
- **`/r/ng/*`** -- Angular primitives

The React registry covers the full catalog; the Angular registry is
still growing. See the [Architecture doc](/platform/architecture) for
the roadmap.

## Source

The registry generator lives in [packages/registry](https://github.com/usegremorie/gremorie/tree/develop/packages/registry)
on GitHub. It walks the workspace, builds an item JSON per primitive,
and writes the index. Generation runs on every push to `develop`.
